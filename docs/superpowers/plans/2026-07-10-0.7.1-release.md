# EmbedAnything 0.7.1 Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the workspace and Maturin-generated Python package report release version `0.7.1`.

**Architecture:** The root workspace manifest provides the version inherited by every
workspace crate. Maturin reads the Python extension crate's inherited version, so no
separate Python metadata version is changed. The root's exact path dependency on
`processors-rs` is kept synchronized with the workspace release version.

**Tech Stack:** Cargo workspace, Cargo metadata, Maturin/Python package configuration.

## Global Constraints

- Release version is exactly `0.7.1`.
- Preserve all pre-existing implementation changes.
- Do not create a Git tag or publish artifacts.
- Do not create a Git commit unless explicitly requested.

---

### Task 1: Synchronize release metadata and validate the packages

**Files:**
- Modify: `Cargo.toml:19`
- Modify: `Cargo.toml:27`
- Verify: `Cargo.lock`
- Verify: `rust/Cargo.toml:1-9`
- Verify: `python/Cargo.toml:1-4`
- Verify: `pyproject.toml:5-29`

**Interfaces:**
- Consumes: Root workspace version in `[workspace.package]`.
- Produces: `embed_anything`, `processors-rs`, `server`, and
  `embed_anything_python` package metadata at version `0.7.1`.

- [ ] **Step 1: Verify the release metadata before the update**

Run:

```bash
cargo metadata --no-deps --format-version 1
```

Expected: The workspace package entries expose the current root-derived version and
the Python package remains configured with `dynamic = ["version"]`.

- [ ] **Step 2: Update the root workspace release version and internal dependency**

Replace these root manifest entries:

```toml
version = "0.7.0"
processors-rs = { path = "processors", version = "=0.7.0" }
```

with:

```toml
version = "0.7.1"
processors-rs = { path = "processors", version = "=0.7.1" }
```

- [ ] **Step 3: Refresh metadata without changing unrelated dependencies**

Run:

```bash
cargo check --workspace
```

Expected: Cargo accepts the synchronized local version constraint. If this command
modifies `Cargo.lock`, retain only lockfile changes made by Cargo as part of the
check.

- [ ] **Step 4: Assert all published package versions**

Run:

```bash
cargo metadata --no-deps --format-version 1 | jq -r '.packages[] | select(.name == "embed_anything" or .name == "processors-rs" or .name == "embed_anything_python" or .name == "server") | "\(.name) \(.version)"'
```

Expected:

```text
embed_anything 0.7.1
processors-rs 0.7.1
embed_anything_python 0.7.1
server 0.7.1
```

- [ ] **Step 5: Verify the Python distribution version source**

Run:

```bash
maturin build --manifest-path python/Cargo.toml --release --out /tmp/embedanything-0.7.1-dist
```

Expected: The generated wheel filename contains `embed_anything-0.7.1`; report
environmental build failures separately from version metadata failures.

- [ ] **Step 6: Inspect the final release diff**

Run:

```bash
git diff --check && git diff -- Cargo.toml Cargo.lock
```

Expected: No whitespace errors; only the expected release metadata and any
Cargo-generated lockfile changes are present in the release diff.
