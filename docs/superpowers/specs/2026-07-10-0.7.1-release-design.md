# EmbedAnything 0.7.1 release preparation

## Scope

Prepare the repository for version `0.7.1` without creating a Git tag or publishing
any artifacts. Existing implementation changes remain untouched.

## Version source

`Cargo.toml` is the single source of truth for the workspace version. The Rust
workspace packages inherit it with `version.workspace = true`, and the Python
package obtains the same version dynamically through Maturin.

The internal `processors-rs` dependency in the workspace root must require
exactly `0.7.1` so packages published from the workspace resolve the matching
release.

## Changes

1. Update the root workspace version to `0.7.1`.
2. Update the root `processors-rs` path dependency constraint to `=0.7.1`.
3. Regenerate the lockfile only if Cargo changes it as a consequence.
4. Do not modify public implementation code, create a tag, or invoke publishing
   workflows.

## Validation

Confirm Cargo reports `0.7.1` for the release packages and run the applicable
workspace and Python-package build checks. Report any pre-existing failures
separately from failures caused by the version update.
